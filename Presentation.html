Builder features

Core Features

🔐 MITM Proxy + SSE

Auto-generated CA certificate. HTTP/HTTPS intercept on :8888. SSE streams captured requests/responses to UI without polling. No Postman plugins needed.

📋 HTML Report Export

Self-contained HTML with syntax highlighting, Dashboard + Detailed View + comments tab.

📊 DDT Dataset Export

dataset.json in DDT contract format — directly into Newman/ddt-runner without modification.

✏️ Manual Builder Mode

Copy-paste from Postman into Builder UI when proxy capture is not needed.

💾 Session Save

execution.json saves full state — pause at any point, resume from the same place. Includes comments and screenshots.

⬆️ Git Sync

Atomic commit of report + execution + dataset to GitHub in one click.

📈 Allure Reporter

Custom Newman reporter generates Allure-compatible XML with history, trends, and attachments.

💬 Execution Comments

Up to 700 chars per test case. Paste screenshots via Ctrl+V — base64 inline. Lightbox with zoom and drag/pan. Not exported to dataset.

📊 Dynatrace Logs

Attach logs to a test case. Structured-log parsing — auto-highlights errors, traces, and timestamps.

🔗 Jira Linking

Link a Jira ticket to the report via the trb-jira tag. Report Viewer displays the link. Multiple tickets separated by comma.

GitHub without a git client

Git Integration

01 Sign in with GitHub — once, all three toolsClick "Sign In" → enter the code at github.com/login/device → done. No passwords, no tokens stored in the app. Session lasts 8 hours. All three tools (Builder, Report Viewer, HTML Viewer) share the same login.
02 Secure access to private repositoriesThe GitHub token is stored encrypted in the cloud (Cloudflare) — the browser never sees it. Private repositories work transparently. Public repos work with no auth at all.
03 Publish to GitHub — no git client, no terminalSelect what to publish: report, execution file, dataset. Click "Sync" — everything is saved to GitHub in one atomic commit. The report link is ready immediately after.
04 Resume an interrupted session"Open from Git" shows a list of previous executions from the repository. One click — Builder restores all test cases with their results. Useful when a testing cycle is interrupted.
05 Share a report instantlyAfter publishing, the "Share" button generates a direct link. The recipient opens the report in the browser immediately — no download required.

Tool #2

Report Viewer

🗂️ Centralised report archiveAll published reports from the GitHub repository in one place. Folder structure with status (passed / failed / skipped) shown next to each file.
🔍 Smart filtersFind any report by app, API endpoint, executor, date, or Jira ticket. Filters combine and apply simultaneously.
🔄 Always up to dateThe list refreshes automatically every 5 minutes. New reports appear after publishing without a page reload.
💾 Local disk repositoryConnect a local reports folder — same filter and browse interface, no GitHub required. Useful for isolated environments or local archives.
🔗 Jira inline in the treeThe linked ticket is visible next to the report file. Click to open Jira — no separate window search needed.
👁️ Preview without downloadingClick a report to render it directly in the browser. Open in a new tab or copy the link to share with a colleague.

Tool #3

Git HTML Viewer

🌐 GitHub HTML URL renderingPaste a GitHub URL (blob or raw) — Viewer fetches the file via raw API and renders it in an iframe. No server needed.
📂 Drag & DropDrag a .html file into the drop zone — instant preview without uploading to GitHub.
🔑 OAuth Sign In (optional)Device Flow via Cloudflare Worker → sessionId in localStorage. Public repositories work without authentication.
🔗 Shareable linksURL is preserved in the address bar. Buttons: New / Copy URL / Open GitHub. The recipient opens the same report.
🔒 iframe sandboxReport renders in an isolated iframe without allow-same-origin. CSP-safe for embedded viewing.
🔁 Builder integrationAfter publishing a report, Builder automatically generates the Viewer link. From final commit to team review — one button, no intermediate steps.
